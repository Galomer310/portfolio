PROJECTS

Here you will find the personal and client projects that I created with each project containing its case study

A TO-DO List

Description : This project is a simple, functional to-do list web application that helps users manage their tasks. It allows users to add, remove, and mark tasks as completed. The user interface is intuitive and user-friendly, with tasks dynamically displayed and updated as users interact with the app.

What I Learned : DOM Manipulation: I improved my skills in working with the Document Object Model (DOM) to dynamically update the web page in response to user actions (e.g., adding or removing tasks). Event Handling: I learned how to handle different user inputs and events, such as button clicks and input field submissions, using JavaScript. Local Storage: I practiced using the browser's local storage to save the to-do list data, so tasks persist even after the user refreshes the page or reopens the app. CSS Styling: I applied CSS to make the interface look clean and responsive, learning more about layout techniques and style adjustments for different screen sizes.

Web Painter App

Description : This project is an interactive painter web application that allows users to create pixel art by selecting colors and applying them to a grid of div elements on the screen. It features a palette with 21 colors, and users can paint by clicking or hovering over the grid, as well as clear the canvas with a button.

What I Learned: Event Handling and Interaction: I improved my understanding of JavaScript event listeners, specifically handling click and hover events to allow users to apply colors to specific grid cells. CSS Grid Layout: I learned how to effectively use CSS Grid to create a flexible and responsive layout for the painting area and color palette. Dynamic Element Creation: I practiced generating HTML elements dynamically with JavaScript, creating a grid of divs that users can interact with. State Management: I implemented simple state management to keep track of the currently selected color and apply it to grid cells. User Interface Design: I worked on improving the user experience by making the application intuitive and visually appealing, with a fixed color palette and a "Clear" button.

Drum Kit Web Application

Description : This project is an interactive drum kit web application that allows users to play drum sounds by pressing specific keys on the keyboard or by clicking on the drum elements displayed on the screen. It simulates a real drum set experience with sounds for each drum element.

What I Learned : JavaScript Event Listeners: I enhanced my skills in handling keyboard and mouse events in JavaScript, making the drum kit responsive to both keyboard presses and mouse clicks. Audio Handling: I learned how to use the HTML audio element and JavaScript to play sound files, mapping each keypress to its corresponding sound. CSS Animations: I experimented with CSS transitions and animations to provide visual feedback when a drum is played, creating an engaging user experience. Keyboard Input Handling: I gained experience in mapping keyboard keys to specific functions, ensuring that each key triggers the correct sound and animation. Responsive Design: I ensured that the layout is visually appealing and adjusts to different screen sizes, so the app works well on both desktop and mobile devices. How It Works: Playing Sounds: Users can press specific keys on their keyboard, each corresponding to a drum sound (e.g., bass, snare, hi-hat), or click on the drum elements to trigger the sounds. Visual Feedback: When a sound is triggered, the associated drum element lights up or shows an animation, providing real-time feedback for the interaction. Key Mapping: Each key is mapped to a different drum sound, and users can play multiple sounds in succession by pressing the keys. Audio Management: The application uses HTML audio elements for each drum sound, which are triggered by JavaScript events.
